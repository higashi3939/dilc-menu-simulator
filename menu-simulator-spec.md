# DILC Menu Simulator — 仕様書

**バージョン**: v8  
**ファイル**: `menu-simulator-v8.html`  
**リファレンス実装**: Canon EOS R5 Mark II のメニュー構成を参考に叩き台を作成  
**実装形式**: 単一HTMLファイル（外部依存なし、`file://` で直接動作）

---

## 1. 概要

デジタル一眼カメラ（DILC: Digital Interchangeable-Lens Camera）の液晶メニュー画面をブラウザ上でシミュレーションするツール。  
静止画モード・動画モードのメニュー遷移、設定値の変更、専用ウィザードUIを再現する。

リファレンス実装として Canon EOS R5 Mark II のメニュー構成を採用しており、公式マニュアル（cam.start.canon）に準拠した項目・選択肢・排他制御を実装している。メニュー定義は JSON で管理されているため、他機種のメニューに差し替えることも可能。

## 6. メインタブ SVGアイコン

各タブはアイコンを**単色シルエット**で表示する（実機準拠）。色は CSS で制御し、SVG 内の色指定は2系統のみ：

| 用途 | 指定 | idle時の値 | active時の値 |
|---|---|---|---|
| 外形（アイコン本体） | `fill="currentColor"` / `stroke="currentColor"` | `#9097a0`（グレー） | `#ffffff`（白） |
| 内側のくり抜き（カメラのレンズ枠内、再生の▶、cfnの液晶部分など） | `fill="var(--icon-bg)"` | `#060a0d`（タブバー背景） | タブカラー |

CSS 側では `.tab-icon` に `color` と `--icon-bg` をセットし、`.main-tab.act .tab-icon` で active 値に切替える。アクティブタブは **背景全体がタブカラーで塗りつぶされ**、アイコンは白抜きで描画される。

### アイコン共通化（静止画／動画）

実機では撮影系タブのアイコンは静止画モード・動画モードで**共通**のため、コード上は静止画タブのアイコンを動画タブにもエイリアスする：

```js
TAB_ICONS.mrec   = TAB_ICONS.shoot;
TAB_ICONS.maf    = TAB_ICONS.af;
TAB_ICONS.mplay  = TAB_ICONS.play;
TAB_ICONS.mcomm  = TAB_ICONS.comm;
TAB_ICONS.msetup = TAB_ICONS.setup;
TAB_ICONS.mctrl  = TAB_ICONS.ctrl;
```

### アイコン形状一覧

| タブID | アイコン形状 | 構成要素 |
|---|---|---|
| `shoot` / `mrec` | カメラボディ | 矩形ボディ + ファインダー突起 + 右上シャッターボタン + 中央レンズ円（くり抜き + 中心ドット） |
| `af` / `maf` | 「AF」テキスト | Arial bold 11px の `AF` 文字のみ（測距点枠は無し） |
| `play` / `mplay` | 再生ウィンドウ | 角丸矩形 + 内側に三角▶（くり抜き） |
| `comm` / `mcomm` | チルダ波線 | M字型（5点）のジグザグ折れ線（電波塔ではない） |
| `setup` / `msetup` | レンチ | 塗りつぶしシルエット |
| `ctrl` / `mctrl` | 円グラフ風ダイヤル | 円 + 右上扇形のくり抜き |
| `cfn` | カメラ背面ビュー | 矩形筐体 + 内側液晶（くり抜き） + 右側操作ボタン3つ（くり抜き） |
| `my` | 5角星 | 塗りつぶし |

```

```

- 行高さ：32px、左右インデント：20px
- **行は左右50%ずつに分割**：項目名は左半分、設定値は右半分の左端から開始（左寄せ）
- カーソル行：左右に 10px のタブカラーボーダー（上下は 1px）
- カーソル行の背景：`rgba(255,255,255, 0.04)`
- カーソル行の文字色：白（`#fff`）
- 動画記録サイズ（`m1_1`）の設定値：テキストではなくSVGアイコン3連（解像度・fps・圧縮）で表示

**CSS 実装**:
```css
.rl{flex:0 0 50%; ...}                  /* ラベル */
.rv{flex:0 0 50%; text-align:left; ...} /* 値 */
```

## 9. 専用ウィザード

### 9.1 動画記録サイズウィザード（`m1_1`）

**起動条件**: 動画モード → MOVIE タブ → Page 1 → `動画記録サイズ` を `Enter`

#### 画面レイアウト

```
┌─ 動画記録サイズ ─────────────────────────┐ ← オレンジ下線
├──────────────────────────────────────────┤ グレー背景
│  Full HD  16:9   29.97 fps              │ ← 現在値1行目（解像度 + fps）
│  標準 Long GOP H (422)   ~200 Mbps      │ ← 現在値2行目（圧縮方式 + Mbps）
├──────────────────────────────────────────┤
│  [FHD]  [29.97P]  [LGOP]                │ ← タブ行（現在選択値のSVGアイコン）
├──────────────────────────────────────────┤
│  設定アイコングリッド（最大2行×6列）      │
├──────────────────────────────────────────┤
│  ↑↓←→:選択  SET:確定  MENU:閉じる       │
└──────────────────────────────────────────┘
```

#### タブ行の表示仕様

タブ行はテキストラベルではなく、**現在選択中の値のSVGアイコン3個**を表示する：

| 位置 | 表示内容 | 使用関数 |
|---|---|---|
| 左 | 現在の解像度アイコン（フィルム枠 + 解像度名） | `mrIconRes(resId, '#fff')` を 42×24 にスケール |
| 中央 | 現在のフレームレートアイコン（白地黒文字ピル `XX.XXP`） | `mrIconFps(fps, '#fff')` を 50×22 にスケール |
| 右 | 現在の圧縮方式アイコン（`Intra`/`LGOP`/`RAW` ラベル + 修飾） | `mrIconComp(iconId, '#fff')` を 50×22 にスケール |

- アクティブタブ: オレンジ下線 + `opacity: 1`
- 非アクティブタブ: 下線なし + `opacity: 0.55`
- クリックでタブ切替（`MRW.tab` を更新して `mrRenderAll()`）

#### 3タブの内容

| タブインデックス | 内容 | グリッド構成 |
|---|---|---|
| 0 = 解像度 | RAW〜Full HD 12種 | 2行×6列固定 |
| 1 = フレームレート | 解像度依存（23.98〜239.76fps） | 最大2行×6列 |
| 2 = 圧縮方式 | 解像度依存（Intra系3 / Long GOP系2 / RAW系2） | 最大2行×6列 |

#### グリッド仕様

- セルサイズ：幅は `(画面幅 - padding) / 6`（等幅）、高さ `52px`（固定）
- セルスタイル：通常 `border:1px solid #1a2a38`、カーソル `border:2px solid #e05040`（オレンジ）、確定選択 `border:1px solid #4a6880`
- SVGアイコン：各タブとも**大サイズ関数**（`mrIconRes` / `mrIconFps` / `mrIconComp`）を `width="100%"` で縮小描画
- アイコン色：常に白基調（カーソル位置のみ薄オレンジ `#f0a090`）

#### 解像度定義（`MR_RES`）

| ID | 日本語 | 解像度 | 縦横比 |
|---|---|---|---|
| `raw` | RAW | 8192×4320 | 17:9 |
| `sraw` | SRAW | 4096×2160 | 17:9 |
| `8kd` | 8K-D | 8192×4320 | 17:9 |
| `8ku` | 8K-U | 7680×4320 | 16:9 |
| `4kd_f` | 4K-D Fine | 4096×2160 | 17:9 |
| `4kd` | 4K-D | 4096×2160 | 17:9 |
| `4ku_f` | 4K-U Fine | 3840×2160 | 16:9 |
| `4ku` | 4K-U | 3840×2160 | 16:9 |
| `2kd_f` | 2K-D Fine | 2048×1080 | 17:9 |
| `2kd` | 2K-D | 2048×1080 | 17:9 |
| `fhd_f` | FHD Fine | 1920×1080 | 16:9 |
| `fhd` | Full HD | 1920×1080 | 16:9 |

#### 解像度アイコン形状（実機のフィルム枠デザイン）

すべての解像度アイコンは**共通の枠デザイン**を持つ：

- viewBox `56×32`、上辺＋右上の縦線（コの字を逆さまにした開いた枠）
- 左側にフィルム穴3つ（縦長の小矩形を等間隔配置）
- 内側に解像度名のテキスト

「Fine」版は枠形状を変えず、テキストを2段組（上：解像度名、下：`Fine`）で表現する。

#### フレームレートアイコン形状（白地黒文字ピル）

- viewBox `56×22`、白塗り角丸矩形（`rx=3`）
- 中央に `XX.XXP` 形式の文字（整数部・小数部・P すべて同サイズ・同ウェイト）
- 例: `23.98P` / `24.00P` / `29.97P` / `59.94P` / `119.88P` / `239.76P`

#### 圧縮方式アイコン形状（ラベル + 修飾の3パターン）

| iconId | ラベル | 修飾 | 意味 |
|---|---|---|---|
| `hi_intra` | `Intra` | `↑+塗り箱`（箱からあふれる矢印） | 高画質イントラ |
| `std_intra` | `Intra` | なし（ラベル中央配置） | 標準イントラ |
| `lt_intra` | `Intra` | `破線輪郭+↓+小塗り箱`（縮小メタファー） | 軽量イントラ |
| `std_lgop` | `LGOP` | なし | 標準 Long GOP |
| `lt_lgop` | `LGOP` | 縮小メタファー | 軽量 Long GOP |
| `raw_std` | `RAW` | なし | RAW（標準） |
| `raw_lite` | `RAW` | 縮小メタファー | RAW（軽量） |

ベース構造：viewBox `56×22`、白地黒文字の角丸矩形ラベル（`rx=1`）。修飾アイコンは右側に隣接して白で描画。

#### 排他制御（実機マニュアル準拠）

排他関係は**ルックアップテーブル + 判定関数**で実装され、4層で制御される。実機マニュアル（[UG-03_Shooting-2_0040.html](https://cam.start.canon/ja/C017/manual/html/UG-03_Shooting-2_0040.html)）に準拠。

##### 排他関係の全体像

```
┌────────────────────────────────────┐
│ メイン記録形式 (m1_3)               │ ← 撮影メニューの値
└────────────────────────────────────┘
              │
              ▼  排他ルール
┌────────────────────────────────────┐
│ 解像度・圧縮方式の有効/無効が決定    │ ← ウィザード内で表示
└────────────────────────────────────┘
              │
              ▼  選択された解像度
┌────────────────────────────────────┐
│ 解像度に応じたfps配列・圧縮方式配列  │ ← MR_FPS / MR_COMP
└────────────────────────────────────┘
```

##### データ層

| テーブル | 内容 |
|---|---|
| `MR_FPS[resId]` | 解像度ごとに利用可能なfps配列（`cinematic` / `normal` / `highspeed` の3カテゴリ） |
| `MR_COMP[resId]` | 解像度ごとに利用可能な圧縮方式配列 |

##### 排他ルール（メイン記録形式 → 解像度・圧縮方式）

**解像度の有効性 (`isResEnabled(resId)`)**

| メイン記録形式 | 有効な解像度 |
|---|---|
| `RAW` | `raw` / `sraw` のみ |
| `XF-HEVC S YCC422 10bit` | `8kd` / `8ku` / `4kd_f` / `4kd` / `4ku_f` / `4ku` / `2kd_f` / `2kd` / `fhd_f` / `fhd` |
| `XF-HEVC S YCC420 10bit` | 上に同じ（HEVC S系の8K対応） |
| `XF-AVC S YCC422 10bit` | 4K以下のみ（8Kは不可） |
| `XF-AVC S YCC420 8bit` | 4K以下のみ（8Kは不可） |

**圧縮方式の有効性 (`isCompEnabled(comp, resId)`)**

| 圧縮方式 (iconId) | 有効になる条件 |
|---|---|
| `hi_intra` (高画質Intra) | メイン記録形式 = `XF-AVC S YCC422 10bit` のときのみ |
| `std_intra` (標準Intra) | 解像度が `8kd` / `8ku`（**8K特例**）、または メイン記録形式 = `XF-AVC S YCC422 10bit` |
| `lt_intra` (軽量Intra) | 同上（8K特例または AVC S 422 10bit） |
| `std_lgop` (標準 Long GOP) | 制限なし（RAW以外のすべての解像度・記録形式で可） |
| `lt_lgop` (軽量 Long GOP) | 同上 |
| `raw_std` (RAW標準) | 解像度が `raw` / `sraw` のときのみ |
| `raw_lite` (軽量RAW) | 同上 |

##### 動的判定関数（取得層）

```js
getMainRecFormat()              // m1_3 の現在値 (日本語側) を取得
isMainRecRaw()                  // メイン記録形式が RAW か
isMainRecHevcS()                // メイン記録形式が XF-HEVC S 系 (8K許可) か
isMainRecAvc422_10()            // メイン記録形式が XF-AVC S YCC422 10bit (Intra系全許可) か
isResEnabled(resId)             // 解像度が現在のメイン記録形式と互換か
isCompEnabled(comp, resId)      // 圧縮方式が現在のメイン記録形式・解像度と互換か

mrFpsAll()                      // 現在解像度で選べるfps配列を全件返す
mrComps()                       // 現在解像度の圧縮方式配列を返す
mrCurrentFps() / mrCurrentComp() // Math.min でクランプして範囲外回避
```

`getMainRecFormat()` はランタイムデータ `DEF`（`SAMPLE` のディープコピー）を参照する。ユーザーが `m1_3` を選択ダイアログで変更すると、その変更が即座に反映される。

##### 自動補正 (`reconcileAfterMainRecChange()`)

メイン記録形式が変更されたとき、現在の選択値が無効になった場合は最初の有効値に自動補正する：

1. 現在の解像度が無効なら、`MR_RES` を先頭から走査して最初の有効解像度に切替
2. グリッドカーソル位置 (`gridRow` / `gridCol`) も新しい解像度の位置に同期
3. 圧縮方式インデックスが範囲外または無効になった場合、最初の有効値へ
4. fpsインデックスが範囲外なら 0 へクランプ

呼び出されるタイミング：

- `dConfirm()` で `m1_3` を確定したとき → さらに `m1_1.mrData` を新しい状態に同期してメニュー行のアイコン3連を更新
- `showMovieRecWizard()` のウィザード起動時（メニューを開いて状態復元した直後にも一回呼ぶ。これにより、メイン記録形式変更後に `m1_1.mrData` の同期が漏れていてもウィザード上で自動補正される）

##### UI表現（無効セルの表示）

灰色化方式で無効を表現：

- セルの `opacity: 0.28`、`cursor: not-allowed`
- 描画は通常通り（アイコン形状は識別可能なまま）
- カーソルは無効セルにも置けるが、Enter/SET は拒否してナビゲーションログに `✗ DENY 選択不可 (メイン記録形式と非互換)` を出力
- `mrApplyGridRes()` 内でカーソル移動時の解像度自動切替を行うが、無効セルではスキップ（カーソル位置だけ更新し `MRW.resIdx` は変更しない）

##### 範囲外回避（クランプ + リセット）

メイン記録形式とは独立した、解像度切替時の不整合回避：

- 解像度を有効セルでクリック確定すると `fpsIdx=0, compIdx=0` にリセット
- `mrCurrentFps() / mrCurrentComp()` は `Math.min(idx, length-1)` でクランプ
- キー操作のラップアラウンドは `mrFpsAll().length` `mrComps().length` で動的にサイズ計算

「クランプ + 変更時リセット + メイン記録形式変更時の自動補正」の三重化で、いかなる操作経路でも不正な選択値にはならない。

#### 状態管理（`MRW`）

```js
const MRW = {
  tab:     0,    // 0=解像度 1=fps 2=圧縮方式
  resIdx:  11,   // MR_RES のインデックス（初期: Full HD）
  fpsIdx:  2,    // mrFpsAll() のインデックス（初期: 29.97）
  compIdx: 2,    // mrComps() のインデックス（初期: 標準 Long GOP）
  gridRow: 1,    // グリッドカーソル行
  gridCol: 5,    // グリッドカーソル列
};
```

#### キー操作

| キー | 動作 |
|---|---|
| `↑` / `↓` | 解像度：行移動、fps/圧縮：項目移動 |
| `←` / `→` | 解像度：列移動（端でタブ切替）、他タブ：タブ切替 |
| `Enter` / SET | 確定・メニューに書き戻し |
| `Esc` / MENU | キャンセル・閉じる |

#### 確定後の書き戻し / 状態復元

- `srcItem.value`：サマリー文字列（例：`Full HD  29.97fps  標準 Long GOP (422)`）
- `srcItem.mrData`：`{ resId, fps, compIconId }` — メニュー行のSVGアイコン表示に使用
- `m1_1` 項目には**初期値として** `{ resId:'fhd', fps:'29.97', compIconId:'std_lgop' }` が設定済み（初回起動時もメニュー行にアイコン3連が表示される）
- ウィザード起動時、`currentItem().mrData` から `MRW` 状態を復元（前回選択値からスタート）

**起動条件**: 動画モード → MOVIE タブ → Page 6 → `タイムラプス動画` を `Enter`

#### 設定項目

| 項目 | 型 | 範囲 |
|---|---|---|
| タイムラプス撮影 | toggle | しない / する |
| 撮影間隔 | 時:分:秒入力 | 00:00:02 〜 99:59:59 |
| 撮影枚数 | 4桁数字入力 | 0002 〜 3600 |
| 動画記録サイズ | select | 6種（8K-U〜Full HD、すべて 29.97fps） |
| メイン記録形式 | select | XF-HEVC S YCC422 10bit / XF-AVC S YCC422 10bit / XF-AVC S YCC420 8bit |

#### メニュー側との連動（`m6_2`）

- メニュー行の値（`する` / `しない`）はウィザードのタイムラプス撮影設定とリアルタイム連動
- ウィザード起動時、メニュー行の現在値（`m6_2.value`）から `TLW.enabled` を復元
- ウィザード内で「タイムラプス撮影」を切り替えると、メニュー行に即時反映（`tlWriteBack()`）

#### 配色とレイアウト

タイムラプスウィザードは MOVIE タブから入るため、**親タブ色（`var(--tc)`）** をアクセントカラーとして使う。

- **通常文字色**: 白（`#fff`）。情報的な補助テキストは `#888`、ガイドは `#666`
- **アクセント色**: `var(--tc, #d07020)` を以下に使用
  - ヘッダータイトル下線（2px）
  - 選択行のボーダー（上下1px・左右10px）
  - 選択中の数字・桁・ラジオマーカー
  - Beep 編集UIの大型数字
- **選択行の背景**: `rgba(255,255,255,.06)`
- **エラー色**: 再生時間が1秒未満のとき `#c05030`

選択行スタイルはメイン側のメニュー行（`.mrow.sel`）と統一（上下1px・左右10pxのボーダー、`box-sizing:border-box` でレイアウトずれ防止）。同じスタイルを以下にも適用:
- メインリストの行
- 動画記録サイズ選択UI（6行のリスト）
- select系編集UI（タイムラプス撮影 / メイン記録形式 / 自動露出 / モニター自動消灯）

#### ヘッダータイトルの動的切り替え

ウィザード上部のタイトルは、**現在の階層を表示**する形で切り替わる:

- 編集していないとき: `タイムラプス動画` / `Time-lapse movie`
- 編集中: 現在のカーソル行のラベル（例: `撮影間隔`、`動画記録サイズ`、`メイン記録形式` 等）

各編集UI内には独自のタイトルバーは持たない（ヘッダーで階層を表示するため重複排除）。

#### 動画記録サイズの表示（アイコン3連）

タイムラプス側でもメイン側 `m1_1` と同じく、動画記録サイズはSVGアイコン3連（解像度＋fps＋圧縮方式）で表示する。データ構造は `TL_RECSIZES` 配列で、各エントリが `{resId, fps, compIconId, ja, en}` を持つ:

```js
const TL_RECSIZES = [
  { resId:'8ku', fps:'29.97', compIconId:'std_intra', ja:'8K-U  29.97fps  標準イントラ',   en:'...' },
  { resId:'8ku', fps:'29.97', compIconId:'lt_intra',  ja:'8K-U  29.97fps  軽量イントラ',   en:'...' },
  { resId:'4ku', fps:'29.97', compIconId:'hi_intra',  ja:'4K-U  29.97fps  高画質イントラ', en:'...' },
  { resId:'4ku', fps:'29.97', compIconId:'std_intra', ja:'...',                              en:'...' },
  { resId:'4ku', fps:'29.97', compIconId:'lt_intra',  ja:'...',                              en:'...' },
  { resId:'fhd', fps:'29.97', compIconId:'std_intra', ja:'...',                              en:'...' },
```

メインリスト行・動画記録サイズ選択UIの両方で、`mrIconResSmall / mrIconFpsSmall / mrIconCompSmall` を流用してアイコン表示。動画記録サイズ選択UIは6行のリストで、各行はアイコン3連のみ（ラジオボタン・テキストなし。選択行は親タブ色のボーダーで判別可能）。

#### リアルタイム計算表示

- **撮影所要時間**：撮影間隔 × 撮影枚数（24時間超は `N days` 形式）
- **再生時間**：撮影枚数 ÷ 29.97fps。1秒未満は赤色で警告

| キー | 動作 |
|---|---|
| `↑` / `↓` | 項目選択 / 編集中は値変更 |
| `←` / `→` | 撮影間隔・撮影枚数編集時：フィールド（桁）移動 |

### 11.4 スクリーンショット機能

液晶画面（`#lcd`、520×346px）の現在の表示を PNG ファイルとして保存する機能。**外部ライブラリ不要**で、ブラウザ標準API（SVG `<foreignObject>` + Canvas API）のみで実装。

#### 起動方法

コントロールパッド右側の📸ボタン（`#btnScreenshot`）をクリック。`takeScreenshot()` 関数が呼ばれる。

#### 実装フロー

1. `#lcd` 要素を `cloneNode(true)` で複製
2. ドキュメント内の全 `<style>` タグを抽出して結合（外部CSSなしの単一HTMLなのでこれで完全再現）
3. `XMLSerializer` でクローンを XHTML 文字列にシリアライズ
4. SVG `<foreignObject>` で囲み、xhtml namespace を明示
5. **data URL** に変換（`'data:image/svg+xml;charset=utf-8,' + encodeURIComponent(svg)`）
6. `<img>` にロード後、Canvas に `drawImage`（`devicePixelRatio` を考慮した高DPI対応）
7. `canvas.toBlob()` で PNG を生成し、`<a download>` でダウンロードトリガー

#### Blob URL ではなく data URL を使う理由

Chrome では Blob URL から生成した SVG 画像は **別オリジン扱い** となり、Canvas が tainted（汚染）状態になって `toBlob()` が失敗する（`SecurityError: Tainted canvases may not be exported.`）。

data URL（`data:image/svg+xml;charset=utf-8,...`）は同一オリジン扱いされるため、tainted を回避できる。

#### 出力ファイル名

`dilc-menu-YYYYMMDD-HHMMSS.png` 形式（タイムスタンプ自動付与）

#### ログ出力

成功時はナビゲーションログに `📸 SS` + ファイル名、失敗時はエラーメッセージを出力。

### v6（メイン記録形式と動画記録サイズの排他制御）

#### 動画モード Page 1 メニュー項目の文言変更

- `m1_5`: 「デュアル撮影（静止画と動画）」→「動画撮影中静止画記録」、選択肢「しない/する」→「OFF/ON」（実機表記準拠）

#### メイン記録形式 ↔ 動画記録サイズの排他制御（実機マニュアル準拠）

実機マニュアル（[UG-03_Shooting-2_0040.html](https://cam.start.canon/ja/C017/manual/html/UG-03_Shooting-2_0040.html)）に準拠した排他関係を実装：

- メイン記録形式が `RAW` のとき、解像度は `RAW` / `SRAW` のみ。それ以外は不可
- メイン記録形式が `XF-HEVC S` 系（YCC422 10bit または YCC420 10bit）のとき、`8K-D` / `8K-U` が選択可能
- メイン記録形式が `XF-AVC S` 系のとき、`8K-D` / `8K-U` は不可
- 圧縮方式の `hi_intra`（高画質Intra）は `XF-AVC S YCC422 10bit` 専用
- 圧縮方式の `std_intra` / `lt_intra`（標準/軽量Intra）は 8K特例または `XF-AVC S YCC422 10bit` のみ
- メイン記録形式変更時、現在の選択値が無効化された場合は最初の有効値に**自動補正**（`reconcileAfterMainRecChange()`）
- ウィザード内の無効セルは `opacity: 0.28` で灰色化、カーソルは置けるが Enter/SET は拒否してナビゲーションログに `✗ DENY` を出力
- 判定ロジックは `isResEnabled(resId)` / `isCompEnabled(comp, resId)` に集約。実装は SECTION 6 (`mrXxx` 関数群)
